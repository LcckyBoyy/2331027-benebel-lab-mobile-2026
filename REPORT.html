1.1 Home Screen — Berita Terkini + Filter Kategori

Halaman utama menampilkan 7 berita terkini dari NewsAPI. Pengguna dapat memfilter berita berdasarkan kategori (General, Tech, Sports, Business, Health, Fun, Science) melalui horizontal chip selector.

2. Organisasi Kode

Struktur Folder

lib/
├── models/         → Data model (Article)
├── services/       → API calls & caching logic
├── providers/      → State management (Provider)
├── views/          → Screens (Home, Search, Detail)
├── widgets/        → Reusable UI components
└── main.dart       → Entry point & theme config

Mengapa Kode Ini Mudah Diperluas?

Arsitektur project mengikuti prinsip Separation of Concerns — setiap folder bertanggung jawab atas satu hal:

Folder Tanggung Jawab Contoh Perluasan
models/ Mendefinisikan bentuk data (Article.fromJson()) Tambah model baru tanpa ubah file lain
services/ Semua API call & caching. UI tidak pernah panggil API langsung Ganti API? Cukup ubah file ini saja
providers/ Jembatan antara service dan view. Kelola state (loading, data, error) Tambah provider baru untuk fitur baru
views/ Murni UI — hanya baca data dari provider Tambah screen baru, pakai widget yang sudah ada
widgets/ Komponen reusable (ArticleCard, LoadingShimmer, AppErrorWidget) Dipakai konsisten di semua screen

Dengan pemisahan ini, menambah fitur baru tidak perlu mengubah kode yang sudah ada — cukup tambahkan file baru di folder yang sesuai.

State Management — Provider

Provider dipilih karena:

  • Alur data eksplisit: Service → Provider → View
  • Direkomendasikan oleh tim Flutter untuk skala project ini
  • Tepat ukuran — tanpa boilerplate berlebih seperti BLoC, dan lebih transparan dari GetX
  • Mudah diuji — logika bisnis terisolasi di layer Provider
